A Random Restaurant Generator using SQLite

Tue 20 November 2012
By Alesha Summers

In misc.

I've always had the problem of not being able to choose what I want for dinner. I decided I would create a small database of restaurants in my area and have SQLite choose one for me randomly. I used this tutorial as a reference.

I created a table for all the restuarants with information like the name of the restaurant, approximate distance from where I live, relative price range, and type of cuisine, since those were the most important factors for me in making a decision. No need for normalization here, just a quick list of restaurants that can be picked from randomly.

CREATE TABLE restaurants(id integer primary key, name text, cuisine text, proximity text, priceRange text);

Then it was time to fill the table with some neighborhood restaurants. For example,

INSERT INTO restaurants(id, name, cuisine, proximity, priceRange) VALUES(1, McDonalds, burgers, short, cheap);

After the table contains some restaurants, it's easy to pick one randomly with the following SQL query

SELECT * FROM Restaurants ORDER BY random() LIMIT 1;

This is an easy way to get a random restaurant from the table, but it can also be extended to exclude values that aren't matching any preferences I happen to have at the moment

A few friends wondered why I didn't just download an app that already exists for this very purpose (e.g., Urbanspoon), but sometimes it's just more fun (not to mention good practice) to build things yourself.
